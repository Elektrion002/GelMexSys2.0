Manual Operativo (Usuario)
1. Proposito del Módulo

Este sistema está diseñado específicamente para automatizar y archivar el proceso de ventas de **Missa Machines**, reemplazando las plantillas de Word. Sirve para capturar datos exactos de cotizaciones de maquinaria (ej. Despulpadoras), estandarizar los cobros y brindar notas profesionales a los clientes.

2. ¿Quién lo usa?

Está destinado para los asesores de venta (ej. Ing. Sergio Misael) y administradores que necesiten realizar o consultar cotizaciones históricas sin interferir en la operación regular de GelMexSys.

3. Directorio de Clientes (Gestión CRM Básica)

No tienes que ingresar los clientes a mano de forma independiente. El sistema está automatizado de la siguiente manera:

  • Al crear una Nueva Venta, el sistema toma los datos de la sección superior y genera automáticamente el registro en la cartera.
  • En la pestaña Clientes del panel lateral, verás la agenda completa con el número de compras históricas asociadas a cada persona/negocio.
  • Diseñé una función de Modificación: Desde ahí existe la opción de *"Editar"*, para corregir errores de dedo en direcciones o teléfonos sin afectar sus recibos previos.
4. Flujo de Trabajo y Documentos Físicos (PDF)
  • **Nueva Venta:** Al presionar "Nueva Venta" el usuario visualizará un formulario... Al teclear el Subtotal y el Anticipo, la aplicación calculará el *Saldo Restante* automáticamente.
  • **Impresión (Nota Física):** Posterior al guardado, serás dirigido a una hoja formato A4 virtual. Da clic en **"Imprimir"**. Si seleccionas "Microsoft Print to PDF" podrás guardar el archivo comercial.
Arquitectura Técnica
1. Estructura Independiente (Arquitectura Modular)

A nivel de servidor, el submódulo fue programado manteniendo el principio de "Single Responsibility". Opera bajo su propio Blueprint de Flask, lo que significa que un error grave en este módulo no impactaría al resto de la aplicación GelMexSys.

  • app/blueprints/missamachines.py Contiene los endpoints y la lógica CRUD (Crear, Leer) principal.
  • app/templates/missamachines Carpeta exclusiva donde reside toda la interfaz aislada (Base Layout, Índice, Venta, Directorio y Docs).
2. Patrón de Base de Datos (ORM SQLAlchemy)

Las tablas necesarias no utilizan ni corrompen las tablas de Clientes de GelMex. Se crearon dos modelos encapsulados:

  • MissaCliente: Tabla dedicada a recolectar nombre de empresa/comprador, teléfono o sector para el negocio de maquinaria.
  • MissaVenta: Relacionada por Foreign Key `cliente_id`, contiene todos los esquemas técnicos configurables de la maquinaria (modelo, voltaje, caballos de fuerza, cilindros y totales).
3. Motor PDF Engine

En lugar de utilizar complejas librerías de Python como *ReportLab* o *pdfkit* que saturan los recursos del servidor (VPS), se desarrolló una solución **Client-Side Rendering**:

Se crearon Media-Queries de formato de estilo @media print. Esto delega la responsabilidad visual al navegador (Chrome/Edge), dictando medidas exactas, ignorando elementos de interfaz extra y generando notas en PDF nativos de altísima resolución en una fracción de milisegundo.
